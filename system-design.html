System Design Roadmap

System Design focuses on building scalable, reliable and high-performance software systems. Interviews test thinking, not perfection.

Scalability · Reliability · Trade-offs

Phase 0: What is System Design?

System Design means creating a blueprint of a large-scale system: components, communication, scaling strategy and failure handling.

  • Components kaunse honge
  • Kaise communicate karenge
  • Scale kaise karega
  • Failure handling

Interview opener: “Design a scalable system for ___”

Phase 1: Basics

  • Client–Server architecture
  • HTTP & REST APIs
  • Status codes & JSON

Har real system isi base pe banta hai.

Phase 2: Scalability

  • Vertical scaling (powerful server)
  • Horizontal scaling (multiple servers)

Real-world systems prefer horizontal scaling.

Phase 3: Load Balancing

  • Traffic distribution
  • High availability
  • Round Robin, Least Connections

“Why do we need a load balancer?” — very common question.

Phase 4: Databases

  • SQL vs NoSQL
  • Read vs Write heavy systems
  • Indexing, Replication
  • Sharding (basic idea)

Phase 5: Caching

  • Client-side cache
  • Server-side cache
  • Redis / Memcached

Cache = performance boost + less DB load.

Phase 6: Consistency & Availability

  • CAP theorem (basic)
  • Consistency vs Availability
  • Partition tolerance

Phase 7: Async Processing

  • Message queues
  • Background jobs
  • Kafka / RabbitMQ (concept)

Phase 8: Reliability & Fault Tolerance

  • Replication
  • Failover
  • Health checks
  • Graceful degradation

Phase 9: Security

  • Authentication & Authorization
  • HTTPS
  • Rate limiting
  • Encryption

Phase 10: High-Level Design (HLD)

  • Services (boxes)
  • Connections
  • Data flow

Freshers ke liye mostly HLD hi poocha jata hai.

Phase 11: Low-Level Design (LLD)

  • Classes
  • APIs
  • Data models

Phase 12: Common Questions

  • URL Shortener
  • Chat Application
  • News Feed
  • Rate Limiter

Final System Design Checklist

Tum system-design ready ho jab tum requirements clarify kar sakte ho, HLD bana sakte ho, caching & DB justify kar sakte ho, aur trade-offs clearly explain kar pao.
